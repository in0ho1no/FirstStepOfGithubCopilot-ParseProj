# セキュリティポリシー

## 安全な使用のためのガイドライン

### 信頼できる入力のみを使用すること

本ツールは `.mtpj` ファイルと、そこから参照されるソースファイルを解析します。
これらのファイルを信頼できないソースから取得した場合、以下のリスクがあります。

- **悪意ある `.mtpj`**: カテゴリ名・ファイル名・マクロ名・値に改行やMarkdown記法を
  埋め込むことで、生成物（`copilot-project-structure.md`）に偽の見出しや指示を
  挿入できる可能性があります（本ツールはサニタイズ処理を実施していますが、
  全ての攻撃ベクターを網羅することを保証するものではありません）。
- **悪意あるソースファイル**: `#include` スキャン時にソースファイルの内容を読み込みます。
  ファイルシステムのパストラバーサルは防御していますが、意図的に大量の
  `#include` を記述したファイルによる処理の遅延は排除できません。

**信頼できないプロジェクトや第三者から受け取ったファイルには使用しないでください。**
使用する場合は、生成された Markdown を GitHub Copilot の指示として利用する前に
内容を目視で確認してください。

### 生成物に含まれる情報

生成される `copilot-project-structure.md` には以下の情報が含まれます。

- プロジェクトのディレクトリ構造とファイル名
- ビルドモード名とマクロ定義（`-D` オプション相当）
- インクルードパス
- ファイル間の `#include` 依存関係

これらはプロジェクトの内部構造を示す情報です。
`.github/` への配置後、リポジトリの公開範囲に応じて外部に露出します。
**公開リポジトリに機密性のあるマクロ定義やパス構造を含むファイルをコミットしないでください。**

本リポジトリの `.gitignore` は `.github/*.md` を除外しています。
生成物を意図的に管理対象とする場合はこの設定を変更してください。

### AI 指示面（prompt injection）への注意

生成される Markdown は GitHub Copilot がコンテキストとして読み込む設計です。
`.mtpj` 内の値が Copilot への指示として解釈されることを意図的に利用した
プロンプトインジェクション攻撃が成立しうることを理解したうえで使用してください。

本ツールは出力時にサニタイズ処理を行いますが、AI モデルの解釈は
テキストの形式だけに依存しないため、完全な無害化を保証できません。
